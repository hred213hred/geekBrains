1. Произвести ручную настройку сети в Ubuntu, на каждом шаге сделать скриншоты.
-------------------------
Скриншоты к сожалению сделать не удалось, поэтому опишу.
На виртуалке был ip- 10.0.2.15
Название сетевого интефейса: enp0s3
Дропаем его: sudo ip addr flush dev enp0s3
Проверяем инет,его нет.
Узнаем какие ip адреса у нас свободны: nmap -v -sP 10.0.2.2/24
Так же узнаем шлюз: 10.0.2.2 (подсвечивает его как _gateway)
Широковещательный адрес сети: 10.0.2.255
Производим настройку:
sudo ip link set enp0s3 up
sudo ip addr add 10.0.2.220/255.255.255.0 broadcast 10.0.2.255 dev enp0s3
Проверяем инет, его нет, что логично, так как мы не указали шлюз
Указываем его: sudo ip route add default via 10.0.2.2
Проверяем инет, он есть.

2. Переключить настройку сети на автоматическую через DHCP, проверить получение адреса.
-------------------------
Дропаем интерфейс: sudo ip addr flush dev enp0s3
Проверяем инет,его нет.
Прописываем в файле /etc/network/interfaces следующее:
"auto enp0s3
iface enp0s3 inet dhcp"
Перезапускаем sudo service networking restart
Проверяем, что получили ip и есть выход в инет:ip addr
ip - 10.0.2.15
Выход в инет есть.

3. Изменить адрес DNS на 1.1.1.1 и проверить доступность интернета, например, открыв любой браузер на адрес https://geekbrains.ru.
-------------------------
Прописываем в файле /etc/network/interfaces следующее:
"dns-nameservers 1.1.1.1"
Проверяем работу инета, он работает.
